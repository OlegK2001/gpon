# Руководство пользователя GPON Network Simulator

## Введение

GPON Network Simulator — интерактивный симулятор GPON-сети для изучения сетевой безопасности и воспроизведения атак в контролируемой среде.

## Основные понятия

### Компоненты сети

- **OLT (Optical Line Terminal)** — оптический линейный терминал провайдера
- **Сплиттеры** — оптические разветвители
- **ONT (Optical Network Terminal)** — терминалы у клиентов
- **CPE Router** — роутеры абонентов
- **CPE Client** — устройства абонентов (ПК, телефоны)
- **Switch** — коммутатор провайдера
- **BRAS/DHCP Server** — серверы управления

### Протоколы

- **OMCI** — управление ONT
- **DHCP** — получение IP-адресов
- **ARP** — разрешение адресов
- **IGMP** — групповая рассылка
- **VLAN** — виртуальные сети

## Быстрый старт

### 1. Запуск симулятора

```bash
docker-compose up -d
```

Откройте http://localhost:3000 в браузере.

### 2. Работа с топологией

- **Создание устройства**: Нажмите "Добавить устройство" в панели инструментов
- **Соединение устройств**: Перетащите линию между устройствами
- **Просмотр устройства**: Кликните на устройство для просмотра деталей

### 3. Запуск сценария атаки

1. Выберите сценарий в левой панели
2. Нажмите "Запустить"
3. Наблюдайте за событиями в логах и метриках

### 4. Мониторинг

- **Метрики**: Отображаются в нижней панели
- **Логи**: Просматривайте в реальном времени
- **Устройства**: Клик на устройство показывает SSH и логи

## Сценарии атак

### DHCP Starvation

**Цель**: Исчерпать пул DHCP-адресов

**Шаги**:
1. Заражение нескольких CPE устройств
2. Генерация массовых DHCP discover
3. Легитимные клиенты не получают IP

**Обнаружение**: Резкий рост использования DHCP-пула

### OMCI Unauthorized Modification

**Цель**: Несанкционированное изменение конфигурации ONT

**Шаги**:
1. Получение доступа к SSH OLT
2. Отправка OMCI-команд на изменение VLAN
3. Перезагрузка ONT

**Обнаружение**: Логи OMCI содержат несанкционированные команды

### ARP MITM

**Цель**: Перехват трафика через ARP poisoning

**Шаги**:
1. Компрометация CPE устройства
2. Отправка поддельных ARP-ответов
3. Перехват трафика жертвы

**Обнаружение**: Изменения в ARP-таблице, подозрительные MAC

## Безопасность

⚠️ **Важно**: Все атаки симулируются в изолированной среде. Не используйте в реальных сетях!

Система предназначена только для образовательных целей в рамках ВКР.
